## 8. 架构设计

### 系统架构

（本项目采用模块化架构）

```
┌─────────────────────────────────────────────────────────┐
│                      用户界面层                          │
│                    (CLI / TUI)                         │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│                      业务逻辑层                          │
│                  (core/* modules)                       │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│                       数据层                             │
│               (db/* + config/* )                       │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│                      外部服务层                          │
│               (OpenClaw API / Provider APIs)            │
└─────────────────────────────────────────────────────────┘
```

### 设计原则

1. **模块化**: 每个功能模块独立，低耦合
2. **可测试**: 业务逻辑与UI分离，便于单元测试
3. **可扩展**: 新功能通过添加模块实现，不修改核心
