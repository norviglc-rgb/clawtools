# ClawTools 安全审计报告（专项）

更新时间：2026-03-27

## 1) 审计范围与方法

- 范围：`clawtools/` 自有代码与脚本（`cli/`, `core/`, `db/`, `scripts/`）
- 不在范围：`node_modules/` 第三方包实现细节
- 方法：静态代码审计 + 关键脚本行为分析 + 构建可行性验证

## 2) 高风险发现（必须优先修复）

### SA-001 配置执行面（对应 CT-003）

- 证据：`core/configurator.ts:77` 使用 `vm.Script` 执行配置内容
- 风险等级：Critical
- 攻击面：本地配置文件污染后可触发任意代码执行
- 建议：
  - 禁止执行型配置加载，只读 JSON
  - 若需兼容旧格式，提供离线迁移而非运行时执行

### SA-002 SQL 拼接（对应 CT-004）

- 证据：`db/index.ts:216` 动态拼接 `UPDATE app_config SET ...`
- 风险等级：Critical
- 攻击面：输入含特殊字符可破坏 SQL 语义
- 建议：
  - 全面参数化 SQL
  - 对外部输入做 schema 校验

### SA-003 破坏性安装命令（对应 CT-001）

- 证据：`scripts/install.ps1:86`, `:89`, `:94`
- 风险等级：Critical
- 攻击面：误删与覆盖风险，造成用户数据不可恢复
- 建议：
  - 删除危险默认行为
  - 删除前增加显式确认与路径保护

## 3) 中高风险发现

### SA-004 自动 `audit fix --force`（对应 CT-002）

- 证据：`package.json:25`, `scripts/install.ps1:110`
- 风险等级：High
- 风险：供应链变化不可控，安装时行为不确定
- 建议：只报告漏洞，不自动强修；修复动作手动触发并审阅变更

### SA-005 密钥加密流程缺少前置校验（对应 CT-005）

- 证据：`cli/screens/config.tsx:33`, `:48`; `core/configurator.ts:110`
- 风险等级：High
- 风险：运行时异常导致配置失败，可能诱发明文旁路方案
- 建议：统一密钥初始化流程 + 显式失败提示 + 禁止空密钥路径

### SA-006 TUI 监听清理策略易误伤（对应 CT-006）

- 证据：`removeAllListeners('data')` 在多个 screen 中使用
- 风险等级：High
- 风险：交互不稳定，可能导致异常退出和不可预期行为
- 建议：集中式输入管理，仅移除当前 handler

## 4) 敏感信息处理评估

- 现状：
  - API Key 设计为加密落库，但加密密钥生命周期管理不完整（见 SA-005）
  - 诊断输出中对环境变量的 KEY 值已做 `[SET]` 遮蔽，方向正确
- 风险：
  - 若加密密钥来源不规范，可能出现不可解密或弱保护
- 建议：
  - 定义统一密钥来源（环境变量优先）
  - 启动时校验密钥强度与存在性
  - 在日志中禁止输出敏感原文

## 5) 安全修复优先级路线

1. 第一阶段（立即）：SA-001, SA-002, SA-003
2. 第二阶段（本迭代）：SA-004, SA-005, SA-006
3. 第三阶段（持续治理）：测试体系、安全回归清单、CI 安全门禁

## 6) 安全回归建议清单

- 检查安装脚本是否仍含破坏性命令
- 检查是否仍存在 `vm.Script` 或等价动态执行
- 检查 SQL 语句是否全部参数化
- 检查 API Key 保存链路是否必须经过密钥初始化
- 检查日志与报错是否泄露敏感信息
