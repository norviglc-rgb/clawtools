# ClawTools 需求说明书

> 本文档是 ClawTools 项目的原始需求来源，定义了项目的目标和功能范围。
> 所有项目决策都应锚定此文档。

## 1. 项目背景

OpenClaw 是一个运行在用户自有设备上的个人 AI 助手，支持多渠道（WhatsApp、Telegram、Discord 等）响应。用户需要一个友好的工具来安装、配置和管理 OpenClaw。

## 2. 核心需求

### 2.1 安装与配置

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 环境检测 | 检测 Node.js、平台、防火墙、公网IP等 | P0 |
| 一键安装 | 支持 stable/beta/dev 版本通道 | P0 |
| 版本选择 | 用户可选 OpenClaw 版本 | P0 |
| 卸载功能 | 干净卸载 OpenClaw | P1 |

### 2.2 配置管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Provider 预设 | OpenAI、MiniMax、智谱(GLM) 等 | P0 |
| API Key 安全存储 | 加密存储到本地 SQLite | P0 |
| 配置模板 | 预置优化配置（付费跳过、推荐项打开） | P1 |

### 2.3 诊断与备份

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 系统诊断 | 检测 OpenClaw 运行状态 | P0 |
| 备份配置 | 打包配置文件 | P0 |
| 恢复备份 | 从备份恢复 | P0 |
| 迁移导出 | 跨平台迁移（Docker/HyperV/原生） | P2 |

### 2.4 文档与安全

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 文档搜索 | 离线搜索 OpenClaw 文档 | P1 |
| 安全扫描 | 防火墙/端口检测（需用户授权） | P1 |

## 3. 技术约束

### 3.1 跨平台支持

必须支持：
- Windows
- Linux (Ubuntu)
- macOS
- WSL
- Docker

### 3.2 技术栈

- Node.js 20+
- TypeScript
- Ink (React-based TUI)
- SQLite (better-sqlite3)
- FlexSearch (搜索)

### 3.3 信任机制

- **全开源**：MIT 许可证
- **逐行注释**：代码即文档
- **隐私授权**：敏感操作需用户显式授权

## 4. 非功能需求

### 4.1 用户体验

- 交互友好（TUI 界面）
- 进度透明
- 操作可回滚

### 4.2 可维护性

- 模块化设计
- 清晰的架构
- 完整的测试覆盖（≥80%）

### 4.3 国际化

- 默认中文
- 预留英文翻译框架

## 5. 未来扩展（v2.0+）

| 功能 | 描述 |
|------|------|
| OpenClaw ACP Skill 安全扫描 | 检测提示词注入和危险操作 |
| 漏洞扫描 | OpenClaw 版本检测和安全漏洞 |
| MCP 接入 | 接入更多外部服务 |

## 6. 成功标准

1. 用户能在 5 分钟内完成 OpenClaw 安装和基本配置
2. 所有功能在三个主要平台（Windows/Linux/macOS）测试通过
3. 测试覆盖率达到 80% 以上
4. 用户可完全信任该工具（开源 + 透明）
