# 05 Release Runbook

更新时间：2026-03-27

本 runbook 采用双轨执行：

- Claude Code 负责自动化修复、构建、验证、发布执行
- Codex 负责关键节点人工复核与发布授权建议

## 1. T-2 ~ T-1：发布前准备

1. 冻结范围
- 只允许处理 `03-launch-backlog.md` 中已定义任务

2. Claude 完成最终实现收敛
- 所有 P0/P1 任务进入可验证状态

3. Codex 进行里程碑审查
- 核对风险台账
- 抽检已完成任务
- 检查文档与代码状态是否一致

4. 补齐证据包
- 验证矩阵
- 测试结果
- 已知问题
- 豁免项
- 回滚步骤

## 2. T-1：发布授权检查

以下条件全部满足才允许进入 T Day：

1. `01-go-no-go-criteria.md` Must 全通过
2. `04-validation-matrix.md` Must Cases 全通过
3. Claude team 当前无 blocked / 无虚假 completed
4. Codex 给出“允许进入发布窗口”的审查结论

## 3. T Day：发布步骤

1. 创建发布分支与标签
2. 执行最终构建与 Windows 主路径 smoke
3. 发布安装脚本与 npm 包
4. 小流量灰度
- 建议 10%
- 或内部先行 + 受控用户组
5. 观察 30-60 分钟关键指标
6. Codex 做灰度后检查
7. 指标正常后全量发布

## 4. 灰度期间必须观察的项目

- 安装成功率
- 配置成功率
- CLI/TUI 异常退出率
- 关键错误日志数量
- 用户是否能看到明确失败原因
- Windows 原生安装是否出现“闪退 / 自动关闭 PowerShell”

## 5. Codex 在发布日的职责

### Checkpoint 1：发布前

- 复核 Must Cases
- 复核回滚入口
- 判断是否允许灰度

### Checkpoint 2：灰度 30 分钟

- 抽查日志、错误码、用户反馈
- 判断是否继续扩量

### Checkpoint 3：全量前

- 给出独立结论：继续 / 暂停 / 回滚

## 6. 回滚预案

1. 停止新版本分发
2. 恢复上一稳定版本安装入口
3. 回退相关脚本/包版本
4. 发布回滚公告
5. 记录事故时间线与根因

## 7. 回滚触发条件

任一满足即执行：

- 出现 P0 安全问题
- 安装成功率连续 15 分钟低于目标阈值
- Windows 主路径大面积失败
- Claude/CI 发现发布包与审查包不一致
- Codex 认定存在不可接受残余风险

## 8. 发布后固定动作

1. D+1：安装与配置漏斗复盘
2. D+3：稳定性与异常复盘
3. D+7：版本复盘与下一轮 backlog
